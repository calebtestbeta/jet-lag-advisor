## Critical Issues

### Issue 1: iOS Safari 深色模式下 Glass 卡片持續顯示深灰

**Severity:** Critical
**Location:** index.html:86–102
**Category:** Visual

**Problem:**
`.lg-glass` 元素在 iOS Safari 系統深色模式下，儘管套用了 `background: rgba(255,255,255,0.96) !important`，卡片仍呈現深灰色。根本原因是 `bg-awake` 漸層（`#bfdbfe → #e0f2fe → #f8fafc`）在 `backdrop-filter: saturate(120%)` 處理後，iOS WebKit 的 backdrop 合成結果和 Chrome 不同，加上 `@media (prefers-color-scheme: dark)` 仍會在 `:root` 寫入深色 CSS 變數，影響 `::before` / `::after` 等 `var()` 引用的元素。

**Impact:**
每次只修補 background 的 opacity，繞不過 backdrop-filter 的 iOS 渲染差異。導致用戶在系統深色模式（手機預設）下看到的 UI 與設計預期完全不同，已反覆出現超過 5 次。

**Recommendation:**
根本修法：在 `body` 加入顯式背景色，讓 backdrop-filter 有穩定的背景可以模糊，不依賴「模糊漸層」的效果。淺色模式時 body 白色，深色模式時 body 深色。

```css
/* Before */
body { margin: 0; font-family: ...; overscroll-behavior: none; }

/* After */
body {
  margin: 0;
  font-family: -apple-system, BlinkMacSystemFont, 'SF Pro Display', 'Segoe UI', sans-serif;
  overscroll-behavior: none;
  background: #f0f6ff; /* 與 bg-awake 基色一致，確保 backdrop 可預測 */
}
@media (prefers-color-scheme: dark) {
  body { background: #0f172a; }
}
```

並將 `.theme-light .lg-glass` 的白色改回更低透明度，依靠穩定的 body 背景而非「幾乎不透明」：

```css
.theme-light .lg-glass {
  background: rgba(255,255,255,0.85) !important;
  backdrop-filter: blur(16px) !important;        /* 移除 saturate 避免 iOS 色調偏移 */
  -webkit-backdrop-filter: blur(16px) !important;
}
```

### Issue 2: datetime-local 輸入框背景近乎全黑

**Severity:** Critical
**Location:** index.html:543–550
**Category:** Visual

**Problem:**
「航程管理」卡片中的兩個日期時間輸入框使用 `lg-glass lg-glass-sm` class，沒有用 `lg-input`。因此它們沒有獲得 `.theme-light .lg-input { background: rgba(255,255,255,0.92) }` 的修正，而是套用未修正的 `.lg-glass` 背景，在深色系統模式下顯示為接近黑色（`rgba(30,30,40,0.60)`）。

**Impact:**
輸入框文字在黑色背景上幾乎不可見（Image #7 中清楚可見）。使用者無法確認已輸入的時間值。

**Recommendation:**

```html
<!-- Before -->
<input type="datetime-local" v-model="tripDepartStr"
  :class="['w-full rounded-xl px-3 py-2 text-sm outline-none lg-glass lg-glass-sm',
    isDark ? 'text-white' : 'text-slate-800']">

<!-- After：改用 lg-input，加 datetime-local 的 min-height 確保 touch target -->
<input type="datetime-local" v-model="tripDepartStr"
  :class="['lg-input', isDark ? 'text-white' : 'text-slate-800']">
```

## Major Issues

### Issue 3: CSS 主題架構層次過多、互相干擾

**Severity:** Major
**Location:** index.html:29–102
**Category:** Visual / Code

**Problem:**
目前控制 `.lg-glass` 外觀的 CSS 有 5 層：
1. `:root` CSS variables
2. `.theme-light` / `.theme-dark` class variables
3. `@media (prefers-color-scheme)` 覆蓋 `:root`
4. `.theme-light .lg-glass { ... !important }` 直接屬性
5. `html[style="color-scheme: only light"]`

這種架構讓每個新修補都可能觸發新的衝突。例如 `@media (prefers-color-scheme: dark)` 設定 `:root { --lg-surface: rgba(30,30,40,0.60) }`，而 `::before` 的 `var(--lg-border-top)` 依然從 `:root` 取值（深色），造成邊框不符預期。

**Recommendation:**
移除 `@media (prefers-color-scheme)` 對 `:root` 的 token 設定（第 56–83 行），僅保留 `.theme-light` / `.theme-dark` class-based tokens。所有 CSS 變數只透過 class 控制，不依賴系統 media query：

```css
/* 移除這段（第 55–83 行）：*/
@media (prefers-color-scheme: dark) { :root { ... } }
@media (prefers-color-scheme: light) { :root { ... } }

/* 只保留 class-based tokens，從根元素繼承 */
.theme-light { --lg-surface: ...; --lg-text-primary: ...; }
.theme-dark  { --lg-surface: ...; --lg-text-primary: ...; }
```

### Issue 4: 捲軸樣式固定白色，不適用淺色模式

**Severity:** Major
**Location:** index.html:275–277
**Category:** Visual

**Problem:**
`::-webkit-scrollbar-thumb { background: rgba(255,255,255,0.3) }` 為白色半透明，在淺色模式下（白色背景）捲軸幾乎不可見。

**Recommendation:**

```css
/* Before */
::-webkit-scrollbar-thumb { background: rgba(255,255,255,0.3); border-radius: 2px; }

/* After */
::-webkit-scrollbar-thumb { background: rgba(128,128,128,0.3); border-radius: 2px; }
.theme-dark ::-webkit-scrollbar-thumb { background: rgba(255,255,255,0.25); }
```

### Issue 6: Urgent Advice 邊框顏色被 !important 蓋掉

**Severity:** Minor
**Location:** index.html:597
**Category:** Visual

**Problem:**
`advice.urgent` 時套用 `border-amber-400/40`，但 `.theme-light .lg-glass { border-color: rgba(0,0,0,0.07) !important }` 會蓋掉這個顏色，urgent 視覺提示無效。

**Recommendation:**
改用 `outline` 而非 `border` 來實現 urgent 邊框，或在 template 用 inline style 強制套用：

```html
:class="['rounded-3xl p-4 lg-glass fade-in']"
:style="advice.urgent ? { outline: '1px solid rgba(251,191,36,0.5)' } : {}"
```

### Issue 8: bg-awake 漸層與 Glass 效果視覺預期不符

**Severity:** Minor
**Location:** index.html:271
**Category:** Visual

**Problem:**
`bg-awake` 使用藍色調 (`#bfdbfe → #e0f2fe`)，與 Glass 白色背景疊加後在 iOS 產生藍灰色調，而非設計中的「白色半透明玻璃」效果。

**Recommendation:**
將 `bg-awake` 漸層調整為更中性的淡藍白色：

```css
/* Before */
.bg-awake { background: linear-gradient(145deg, #bfdbfe 0%, #e0f2fe 45%, #f8fafc 100%); }

/* After：降低藍色飽和度，接近純白 */
.bg-awake { background: linear-gradient(145deg, #dbeafe 0%, #eff6ff 45%, #f8fafc 100%); }
```

## Suggestion

### S1: 統一使用 CSS 變數控制文字顏色

**Location:** index.html (全域 template)

目前 template 中大量使用 `isDark ? 'text-white' : 'text-slate-xxx'` 的重複模式，既難以維護，也不一致（有的用 `text-slate-500`，有的用 `text-slate-600`，有的用 `text-slate-700`）。

已定義好的 `--lg-text-primary`、`--lg-text-secondary`、`--lg-text-tertiary` CSS 變數幾乎沒被 template 使用，卻被重複宣告了 4 次（`:root`、`.theme-light`、`.theme-dark`、`@media`）。

建議：逐步將 `isDark ? ... : ...` 的顏色切換，改用 CSS 變數 + Tailwind 的 arbitrary value，或透過一層 CSS class 統一管理。
