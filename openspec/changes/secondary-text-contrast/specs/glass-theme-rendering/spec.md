## MODIFIED Requirements

### Requirement: 文字對比度
系統 SHALL 確保所有卡片內的標籤與輔助文字符合 WCAG AA 標準（對比比率 ≥ 4.5:1），包含主要文字與次要/輔助文字。

#### Scenario: 時鐘卡片標籤
- **WHEN** 使用者在淺色模式查看出發地 / 目的地時鐘卡片
- **THEN** 「出發地」「目的地」標籤文字可清楚辨識（使用 `text-slate-600` 或更深）

#### Scenario: 時鐘卡片時區偏移標籤
- **WHEN** 使用者在淺色模式查看時鐘卡片下方的時區偏移文字（如「UTC+9」）
- **THEN** 時區文字可清楚辨識（使用 `text-slate-600` 或更深，對比度 ≥ 4.5:1）

#### Scenario: 下拉清單次要資訊
- **WHEN** 使用者在淺色模式查看機場搜尋下拉清單
- **THEN** 國家名稱和 IATA 機場代號等次要資訊可清楚辨識（使用 `text-slate-600` 或更深）

#### Scenario: 行程卡片輔助標籤
- **WHEN** 使用者在淺色模式查看航程管理卡片
- **THEN** 「出發時間」「預計抵達」「距抵達目的地」等標籤文字可清楚辨識（對比度 ≥ 4.5:1）

#### Scenario: urgent 建議卡片邊框
- **WHEN** 某條建議標記為 `urgent`
- **THEN** 卡片邊框顯示琥珀色提示，不被其他 CSS 規則覆蓋

#### Scenario: 建議卡片輔助說明
- **WHEN** 使用者在淺色模式查看時差建議列表
- **THEN** 建議卡片內的輔助說明文字（如「約需 N 天完全適應」）可清楚辨識（對比度 ≥ 4.5:1）
